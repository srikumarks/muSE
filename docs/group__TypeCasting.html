muse_cell fn_string ( muse_env *  env,
void *  context,
muse_cell  args 
)
1 (string thing)

Behaves almost the same as format when applied to thing. The difference is that format accepts any number of arguments whose strings forms are concatenated into one big string. string only accepts one argument.

So why have a separate function?

When thing is a number, string takes an optional second argument that gives a printf style format specification string for converting the number to a string. For everything else, the second argument, if supplied, is ignored. The formatting should not result in generating a string that is longer than 128 characters, including the terminating null character.

References fn_format(), MUSE_FLOAT_CELL, muse_float_value(), MUSE_INT_CELL, muse_int_value(), muse_mk_ctext(), MUSE_NIL, MUSE_TEXT_CELL, and muse_text_contents().

muse_cell fn_format ( muse_env *  env,
void *  context,
muse_cell  args 
)
1 (format ...args...)

Converts each arg to a string, concatenates all the strings and returns the result as a single string. Things that support the 'frmt' view such as bytes? and object? convert themselves to strings.

Objects, in particular, get their "as-string" methods invoked to get a string representation. The "as-string" can also be a field. If it isn't a method, its value is used as the string representation of the object. In particular, the "as-string" implementation can return or be any object which can be converted to a string, such as int, float or even another object.

For example -

1 > (define ex
2  (object ()
3  'name "telepathic-robot"
4  'id 3
5  'as-string (fn (self) (format self.name "-" self.id))))
6 > (format "object=" ex)
7 object=telepathic-robot-3
