1 (image-properties path-to-image)

Creates an object using which you can access the metadata of the given image file. The returned object supports get.

The metadata can be accessed either by symbolic key or by numeric tag code, as specified in the Microsoft documentation.

  • Image property tag constants
  • Image property tag type constants

The symbolic names of the tags are derived from the constant declarations as follows -

  1. A name of the form PropertyTagGpsDestDistRef becomes gps-dest-dist-ref i.e. the PropertyTag prefix is dropped and the camel case is converted into Scheme convention.
  2. For names with abbreviations in them such as PropertyTagJPEGQuality, the lower case form preserves the abbreviation - so PropertyTagJPEGQuality becomes jpeg-quality
  3. YCbCr is also converted into lower case as ycbcr - so PropertyTagThumbnailYCbCrCoefficients becomes thumbnail-ycbcr-coefficients

The following extra properties are also exposed -

'size
1 {vector width-pixels height-pixels}
'physical-size
1 {vector physical-width physical-height}
'resolution
1 {vector hres vres}
'path
1 "path/to/file.jpg"

image-properties:invalid-tag
Handler
1 (fn (resume 'image-properties:invalid-tag props code/tag) ...)
When A property with an invalid tag is requested. The tag can be numeric or symbolic. If the tag is not a number or a symbol, then this exception means that you have to pass a number or a symbol. props is the properties objects which raised the exception. (i.e. this exception is not raised by image-properties itself, but by the object it returns.)
Consequence Cannot retrieve property.
Resume By passing the expected result.
