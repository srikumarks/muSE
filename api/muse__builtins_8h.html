muse_cell  args

1 (substring str start-index [count])

Gives the substring of length count characters of the given string starting from start-index. If the start-index is considered to be modulo [0,length) ... which means you can get the last 3 characters of a string like this -

1 (substring str -3 3)

If you request more characters than is available in the string, you get as much of the string as is avaiable.

The count is optional. If omitted, the substring is considered up to the end of the string. You can use this to shorten the above example to -

1 (substring str -3)

1 (list-files [pattern])

Returns a list of files that patch the given pattern. For example:

1 (list-files "*.jpg")

will list the JPG files in the current folder. Note that the returned list only has the file names and not the full paths to the files.

Another use for list-files is to check whether a particular file exists. If p is the full path to the file, say, then

1 (list-files p)

will be () if the file doesn't exist and will be a single-entry list of the file's name (stripped of path) if the file exists.

Returns a list of folder names for all the folders that exist under the given parent folder. For example:

1 (list-folders "../")

will list the folders above the current folder. Note that the returned list only has the folder names and not the full paths to the folders. Also, the folder names don't end with '/' or any such path separator character.

muse_cell fn_split ( muse_env *  env,
void *  context,
muse_cell  args 
)
1 (split "one;two;;three;" ";") -> ("one" "two" "" "three" "")
1 (split "one=1;two=2;three=3" "=;") -> ("one" "1" "two" "2" "three" "3")

The first argument is the string to split and the second argument is a character separator.

You can define a recursive splitter like this -

1 (define (split-rec str (sep . seps))
2  (if seps
3  (map (fn (s) (split-rec s seps)) (split str sep))
4  (split str sep)))

so that

1 (split-rec "a=1&b=2&c=3" '("&" "="))

will give you

1 (("a" "1") ("b" "2") ("c" "3"))

.

muse_cell fn_windows_registry_entry ( muse_env *  env,
void *  context,
muse_cell  args 
)
1 (windows-registry-entry section entry)

Looks up the given entry path in the given section. The entry's components are to be separated by backslashes. Integers become muSE integers, strings become muSE strings, a multi-string becomes a list of strings and a binary blob becomes a byte array.

Parameters
sectionCan be one of "HKEY_CLASSES_ROOT", "HKEY_CURRENT_CONFIG", "HKEY_CURRENT_USER", "HKEY_LOCAL_MACHINE", "HKEY_PERFORMANCE_DATA", "HKEY_PERFORMANCE_NLSTEXT", "HKEY_PERFORMANCE_TEXT" or "HKEY_USERS"
entryA backslash separated sequence of subkeys.
Example
To get the installed location of muvee Reveal's MVRT.dll, you can query the following registry entry -
1 (windows-registry-entry
2  "HKEY_LOCAL_MACHINE"
3  "SOFTWARE\\Classes\\CLSID\\{30A8E5D8-A4BD-4038-981E-AD2B7AD781EA}\\InprocServer32\\")
