Detailed Description

muSE implements an exception mechanism that allows you to normally resume computation from the point where an exception is raised.

An expression that might raise an exception is wrapped by a try expression and a list of handlers to try.

When a try expression is evaluated, the handlers are all evaluated first and placed on a stack of handlers to try when an exception is raised. (The handler stack is maintained as the value of the internal symbol "{{trap}}".) The try protected expression is then evaluated. When evaluation is complete, the handler stack is unwound and the set of handlers defined by the enclosing try block take effect.

If you want to try an alternative code path instead of raising a resumable exception, use retry instead of raise. Alternative code paths are to be considered part of the "interface" of a function and must be documented along with the function.

typedef struct _resume_point_t resume_point_t

Captures all the information required to jump back to an execution point within stack scope.

Its essentially a cheaper continuation that cannot be retained beyond its marked scope.

typedef struct _trap_point_t trap_point_t

A trap point is a marker for the beginning of a (try...) block.

When you return to a trap point, you return with a value that is supposed to be the value of the try block. Trap points are maintained as a stack of values of the built-in symbol "{{trap}}", which is available via MUSE_TRAP_POINT.
